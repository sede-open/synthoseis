# Synthoseis Web Visualization Dashboard

A static React 18 web application for browsing and interactively slicing 3-D seismic volumes generated by the [Synthoseis](../README.md) pipeline.

---

## Architecture

```
webapp/
  index.html              – root HTML; applies bp5-dark to <body>
  vite.config.ts          – Vite build config; base: "./"
  package.json            – npm deps and build scripts
  tsconfig.json           – TypeScript config (ES2020, bundler resolution)
  src/
    config.ts             – MANIFEST_URL, ZARR_BASE_URL constants
    types/manifest.ts     – VolumeInfo, ManifestEntry TypeScript interfaces
    main.tsx              – ReactDOM.createRoot mount
    App.tsx               – hash-based router
    hooks/
      useManifest.ts      – fetch + parse manifest.json
      useZarrSlice.ts     – zarrita.js openConsolidated(FetchStore) + LRU-5 slice cache
    components/
      ProjectDashboard.tsx – card grid of all runs
      RunCard.tsx          – single run summary card
      RunViewer.tsx        – run inspector with slice viewer
      VolumeSelector.tsx   – Blueprint Select2 grouped by subdir
      SliceViewer.tsx      – react-plotly.js heatmapgl slice viewer
      ColormapSelector.tsx – colormap picker with per-group defaults
      MetadataPanel.tsx    – Parameters + Volumes metadata tabs
```

## Prerequisites

- **Node.js ≥ 20** (tested with v25) and **npm ≥ 10**
- **Python 3.11** with the synthoseis virtual environment activated

## Step 1 — Run the Pipeline

Generate zarr v3 outputs with synthoseis:

```bash
# From the repository root
source .venv/bin/activate
python main.py --config config/example.json --num_runs 1 --run_id seismic_example
```

This writes run outputs under `<project_folder>/seismic__<datestamp>_seismic_example/`.

---

## Step 2 — Generate `manifest.json`

Scan the project folder for completed runs and write a `manifest.json` at its root:

```bash
python scripts/generate_manifest.py \
  --project-folder /path/to/synthoseis_example

# To regenerate an existing manifest:
python scripts/generate_manifest.py \
  --project-folder /path/to/synthoseis_example \
  --overwrite
```

**What the script does:**
- Matches subdirectories named `seismic__<datestamp>_<run_id>/`
- For each run, scans `seismic/`, `geology/`, `horizons/`, `closures/` and root-level QC stores (`*.zarr`)
- Opens each store with `zarr.open_consolidated(path, zarr_format=3)` + `xr.open_zarr(path)` to extract variable name, shape, dtype, dims, chunks, compressor, and attrs
- Reads `parameters.db` (SQLite, table `model_parameters`) for key/value run parameters
- Writes `manifest.json` as a JSON array of run entries
- Missing `parameters.db` → `parameters: {}` (warning logged, no crash)
- Corrupted/unreadable zarr stores are skipped with a warning

### Manifest schema example

```json
[
  {
    "run_id": "seismic_example",
    "folder": "seismic__20260517_seismic_example",
    "datestamp": "20260517",
    "cube_shape": [300, 300, 1250],
    "volumes": [
      {
        "name": "seismicCubes_RFC_7_degrees_20260517",
        "store_path": "seismic/seismicCubes_RFC_7_degrees_20260517.zarr",
        "variable": "amplitude",
        "group": "Seismic",
        "shape": [300, 300, 1250],
        "dtype": "float32",
        "dims": ["inline", "crossline", "time"],
        "chunks": [1, 300, 128],
        "compressor": "blosc:zstd:5:bitshuffle",
        "attrs": {"angle_deg": 7, "sample_rate_ms": 4}
      }
    ],
    "parameters": {
      "cube_shape": "[300, 300, 1250]",
      "incident_angles": "[7, 15, 24]"
    }
  }
]
```

```bash
cd webapp
npm install

```

## Step 4 — Serve the App

Copy (or symlink) `manifest.json` and the run folder(s) into `dist/`, then serve:

```bash
# Simple: copy manifest and run data into dist/
cp /path/to/synthoseis_example/manifest.json webapp/dist/
cp -r /path/to/synthoseis_example/seismic__*/  webapp/dist/

# Serve
python -m http.server 5173 --directory webapp/dist
```

Open **http://localhost:5173** in Chrome 120+ or Firefox 120+.

> **No backend required.** The webapp reads `manifest.json` and fetches zarr chunk files
> directly via HTTP range requests. Any static file server (nginx, Caddy, GitHub Pages, etc.) works.

## Default Colormaps by Group

| Group     | Default colormap |
|-----------|-----------------|
| Seismic   | `RdBu`          |
| Geology   | `Greys`         |
| Closures  | `Hot`           |
| Horizons  | `Viridis`       |
| QC        | `Viridis`       |

The user can override the colormap at any time via the colormap selector.

## Slice Types

| Tab          | Axis fixed | Resulting shape              |
|--------------|------------|------------------------------|
| Inline       | inline     | `[crossline, time]`          |
| Crossline    | crossline  | `[inline, time]`             |
| Timeslice    | time       | `[inline, crossline]`        |
| Horizon slice| horizon    | `[inline, crossline]`        |

> The third tab is labelled **"Horizon slice"** when `VolumeInfo.dims[2] === "horizon"`
> (i.e. for `horizons/depth_maps.zarr`), otherwise **"Timeslice"**.

**Crossline debounce:** the crossline slider only fires on `mouseUp` / `touchEnd`
to avoid flooding the HTTP server with chunk requests mid-drag. A `Spinner` is
shown while the slice is loading.

## Blosc Bitshuffle — WASM Codec Note

The synthoseis pipeline writes zarr v3 stores compressed with
`BloscCodec(cname="zstd", clevel=5, shuffle=BloscShuffle.bitshuffle)`.
zarrita.js decodes chunks via `@zarrita/codecs` (numcodecs-wasm ≥ 0.3),
which supports Blosc + bitshuffle.

**If you encounter a WASM decode error in the browser console** (e.g.
`UnknownCodecError: blosc` or a bitshuffle-specific failure), apply this
one-line fallback in `datagenerator/output_writer.py`:

```python
# Change bitshuffle → shuffle (byte-level, wider WASM support):
_DEFAULT_COMPRESSOR = BloscCodec(
    cname="zstd",
    clevel=5,
    shuffle=BloscShuffle.shuffle,   # was: BloscShuffle.bitshuffle
)
```

Then re-run the pipeline and regenerate `manifest.json`. Byte-level shuffle
still provides good compression ratios for float32 seismic data (≈ 2–3×).

## Development

```bash
cd webapp
npm install
npm run dev          # Vite dev server with HMR on http://localhost:5173
```

Place a `manifest.json` in `webapp/` (or point `MANIFEST_URL` to an external URL)
and the run data accessible from the dev server root.
